M3U vs M3U8: What's the Difference?

December 8, 2025 6 min read

TL;DR: M3U and M3U8 are essentially the same format. The "8" in M3U8 indicates UTF-8 encoding for international character support. Both work with IPTV apps and Strim handles both seamlessly.

What is an M3U File?

M3U stands for "MP3 URL" or "Moving Picture Experts Group Audio Layer 3 Uniform Resource Locator." Despite the name, M3U files aren't limited to MP3 audio - they're used for any media playlist.

An M3U file is simply a text file containing a list of media file locations (URLs or file paths). It tells your media player where to find and play content.

M3U File Structure

#EXTM3U
#EXTINF:-1 tvg-id="CNN" tvg-name="CNN" group-title="News",CNN HD
http://example.com/stream/cnn.ts
#EXTINF:-1 tvg-id="ESPN" tvg-name="ESPN" group-title="Sports",ESPN HD
http://example.com/stream/espn.ts
#EXTINF:-1 tvg-id="HBO" tvg-name="HBO" group-title="Movies",HBO HD
http://example.com/stream/hbo.ts

Key elements:

  • #EXTM3U - Header that identifies the file as an extended M3U
  • #EXTINF - Extended information about the following entry
  • tvg-id - Channel ID for EPG matching
  • tvg-name - Display name for the channel
  • group-title - Category/group the channel belongs to
  • URL - The actual stream location

What is M3U8?

M3U8 is simply an M3U file encoded in UTF-8. The "8" refers to UTF-8 character encoding.

M3U (Original)

  • Uses Latin-1 (ISO-8859-1) encoding
  • Limited to Western European characters
  • Older, original format
  • File extension: .m3u

M3U8 (UTF-8)

  • Uses UTF-8 encoding
  • Supports ALL languages and characters
  • Modern, recommended format
  • File extension: .m3u8

Why Does Encoding Matter?

Character encoding determines how text characters are stored in a file. This matters when your playlist contains:

  • Channel names in non-English languages (Chinese, Arabic, Russian, etc.)
  • Special characters (accents, symbols)
  • Emoji in channel descriptions
  • International group names

M3U8 and HLS Streaming

There's another use for M3U8 that causes confusion: HLS (HTTP Live Streaming). Apple's HLS protocol uses M3U8 files as manifests for adaptive streaming.

Important: An IPTV M3U8 playlist (list of channels) is different from an HLS M3U8 manifest (streaming segments). Both use .m3u8 extension but serve different purposes.

HLS M3U8 Manifest Example

#EXTM3U
#EXT-X-VERSION:3
#EXT-X-TARGETDURATION:10
#EXT-X-MEDIA-SEQUENCE:0
#EXTINF:10.0,
segment_0000.ts
#EXTINF:10.0,
segment_0001.ts
#EXTINF:10.0,
segment_0002.ts

This HLS manifest tells a player to download and play video segments in sequence. It's not a list of TV channels - it's the streaming format itself.

Which Should You Use?

Recommendation: Use M3U8

M3U8 (UTF-8) is the modern standard. It handles all languages and special characters correctly. There's no downside to using M3U8 - all apps that support M3U also support M3U8.

When to use M3U (Latin-1):

  • You're working with very old software that only supports Latin-1
  • Your playlist only contains ASCII characters (A-Z, 0-9)
  • You have specific compatibility requirements

How Strim Handles Both

Strim automatically handles both M3U and M3U8 formats:

  • Automatically detects file encoding
  • Properly displays international characters
  • Exports as UTF-8 (M3U8) by default
  • Preserves special characters in channel names and groups

You don't need to worry about the format - just load your playlist and Strim handles the rest.

Conclusion

M3U and M3U8 are functionally the same playlist format. The only difference is character encoding (Latin-1 vs UTF-8). For IPTV use, prefer M3U8 for better international character support. Both formats work with all modern IPTV apps and Strim.
